# PPT Design_1 — Flat Vector / Clean Illustration

여러 파트(Part)에 걸쳐 제작되는 프레젠테이션에서 **디자인 토큰을 고정**하여 시각적 일관성을 유지하기 위한 스킬입니다. 실제 PPTX 파일을 만들 때는 이 스킬로 규칙을 정하고, `/mnt/skills/public/pptx/SKILL.md`(pptx 스킬)로 빌드하세요.

## 언제 이 스킬을 쓰는가

- "ppt Design_1" 또는 이 문서에 정의된 스타일로 슬라이드를 만들어 달라는 요청
- 이미 이 스타일로 만든 덱의 "Part 2", "다음 파트", "후속 슬라이드"를 제작할 때 → **디자인 토큰을 절대 바꾸지 말고 이 문서 값을 그대로 재사용**
- 사용자가 별도 언급 없이 스타일을 바꿔달라고 하지 않는 한, 이미 이 스킬로 시작된 덱에는 항상 동일한 토큰 적용

## 디자인 토큰 (고정값 — 파트 간 절대 변경 금지)

### 배경 & 캔버스
| 항목 | 값 |
|---|---|
| 배경색 | `#FFFFFF` (순백) |
| 여백 | 넉넉하게 — 슬라이드 가장자리 최소 60px(4:3) / 80px(16:9) 여백, 콘텐츠 밀도 낮게 |
| 슬라이드당 불릿 | 최대 5개 |

### 타이포그래피
| 요소 | 스타일 |
|---|---|
| 제목(Title) | Sans-serif, **Bold**, 슬라이드당 1개, 짧고 명사형으로 종결 |
| 본문(Body) | Sans-serif, Regular, 불릿 형태 |
| 권장 폰트 | Pretendard, Noto Sans KR, 또는 시스템 기본 산세리프(Helvetica/Arial 계열) — 영문/국문 혼용 시 두 폰트 패밀리를 통일 |

### 포인트 컬러 (강조용, 회색·검정 텍스트 기본 위에 포인트로만 사용)
| 이름 | Hex | 용도 |
|---|---|---|
| Deep Blue | `1E3A8A` | 핵심 키워드, 주요 데이터/그래프 강조 |
| Deep Orange | `C2410C` | 경고·리스크·주의 포인트 |
| Light Pink | `F9A8D4` | 보조 강조, 아이콘/도형 배경 |
| Green | `15803D` | 긍정 지표, 성장·완료 상태 |
| 기본 텍스트 | `1A1A1A` (거의 검정) | 본문 텍스트 |
| 보조 텍스트/선 | `6B7280` | 캡션, 구분선, 비강조 요소 |

> hex 값에 `#`를 붙이지 않는 것은 pptxgenjs 색상 파라미터 규칙(다른 스킬의 관행)과 맞춘 표기입니다. 실제 코드에서 `#`가 필요한 라이브러리라면 프리픽스만 붙여 사용하세요.

### 일러스트 스타일
- Flat Vector, Clean Illustration — 그림자 최소화, 단순한 도형 기반 아이콘/일러스트
- 그라디언트·과도한 텍스처 지양, 단색 면 + 포인트 컬러 조합
- 아이콘은 라인 또는 플랫 실루엣 중 하나로 슬라이드 전체 통일

## 텍스트 작성 규칙 (Crucial — 반드시 준수)

**모든 텍스트는 "명사형 종결" 또는 "단문형(개조식)"으로 작성합니다.**
- ❌ "이 기술은 비용을 절감합니다" (완전한 서술형 문장)
- ✅ "비용 절감" / "원가 30% 절감"
- 종결 어미(-다, -함, -습니다) 대신 명사 또는 명사구로 마무리
- 불릿은 한 줄 내외로 압축, 접속사·조사 최소화
- 제목도 동일 규칙 적용 (예: "시장 현황 분석" O, "시장 현황을 분석해보겠습니다" X)

## 레이아웃 원칙

1. 슬라이드당 메시지 1개 — 정보 밀도를 낮추고 여백으로 호흡 확보
2. 불릿 5개 초과 시 슬라이드 분할 또는 하위 그룹으로 재구성
3. 포인트 컬러는 슬라이드당 1~2개 요소에만 사용 (남용 시 강조 효과 상실)
4. 표/차트 사용 시에도 배경은 흰색 유지, 데이터 강조 요소에만 포인트 컬러 적용

## 파트 간 일관성 체크리스트

새 파트 제작 시 아래를 이전 파트와 대조해 확인:

- [ ] 배경 `#FFFFFF` 동일
- [ ] 제목/본문 폰트 패밀리 동일
- [ ] 포인트 컬러 4종 hex 값 동일
- [ ] 불릿 개수 규칙(최대 5개) 준수
- [ ] 텍스트가 명사형/개조식으로 통일
- [ ] 일러스트 스타일(Flat Vector, 그림자 없음) 동일

## 실제 빌드 워크플로우

1. 이 문서의 토큰을 확정(사용자가 값 수정 요청 시 이 문서를 업데이트하고 이후 모든 파트에 반영)
2. 콘텐츠 목차/스토리라인 확정
3. `/mnt/skills/public/pptx/SKILL.md`의 pptxgenjs 워크플로우로 실제 PPTX 생성 — 색상은 위 표의 hex 값을 그대로 사용
4. LibreOffice headless PDF 변환 + `pdftoppm`으로 시각 QA (배경 흰색 여부, 포인트 컬러 정확도, 불릿 개수 확인)

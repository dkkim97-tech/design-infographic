# Claude Code PPT — 파워포인트 없이 PPT 만들기

Claude Code(또는 claude.ai)만으로 PPT를 처음부터 끝까지 만드는 워크플로우입니다.
실제 PPTX 파일 생성은 **Anthropic 공식 `pptx` 스킬**(세션 내장, 별도 설치 불필요)이 담당하고,
이 스킬은 그 앞뒤 단계를 표준화합니다.

## 워크플로우

### 1단계 — 콘텐츠 구조 설계 (꾸미기 전에 내용부터)
- 사용자의 주제·원고·강의자료에서 핵심 메시지를 추출
- 슬라이드 장수와 목차(스토리라인)를 먼저 제안하고 사용자 확인을 받은 뒤 제작 시작
- 슬라이드당 메시지 1개 원칙, 텍스트는 개조식으로 압축

### 2단계 — PPTX 빌드
- 공식 `pptx` 스킬의 워크플로우로 실제 .pptx 파일 생성 (16:9 기본)
- 디자인 스타일 지정이 없으면: 화이트 배경 + 산세리프 + 포인트 컬러 1~2색의 미니멀 스타일
- 이 저장소의 `ppt-design-1` 스킬 스타일 요청 시 해당 디자인 토큰을 그대로 적용
- 도식화: 프로세스/비교/구조는 텍스트 나열 대신 도형 기반 다이어그램으로 변환
- 수치화: 데이터는 표 또는 네이티브 차트로 표현, 핵심 수치는 크게 강조

### 3단계 — 이미지 삽입
- 이미지가 필요한 슬라이드는 위치·크기를 잡은 "이미지 슬롯"으로 먼저 배치
- 사용자가 외부 도구(Flow/나노바나나, 미드저니 등)로 생성한 이미지를 받으면 슬롯에 삽입
- 이미지 없이도 완성본이 되도록 슬롯에는 대체 도형/배경색을 넣어 둠

### 4단계 — 검수 및 후편집 준비
- LibreOffice headless로 PDF 변환 후 슬라이드별 렌더링을 확인하는 시각 QA 수행
  (텍스트 넘침, 겹침, 컬러 오류 확인)
- 완성된 .pptx는 PowerPoint·Google Slides·Canva에서 개별 요소 편집 가능
  (Canva: 만들기 → 가져오기로 .pptx 업로드)

## 원칙

- 코딩 지식 없이 대화만으로 진행 — 사용자에게 코드나 명령어 실행을 요구하지 않음
- 템플릿 탐색에 시간을 쓰지 않음 — 내용에서 출발해 디자인은 규칙으로 자동 결정
- 17장 이상 대형 덱도 목차 확정 → 파트별 생성 → 일관성 체크 순서로 제작
- 수정 요청은 슬라이드 번호 기준으로 받아 해당 슬라이드만 재생성

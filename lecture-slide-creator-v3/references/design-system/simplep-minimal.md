# SimpleP 미니멀 디자인 시스템

> 업로드된 SimpleP_PowerPoint 템플릿 분석 기반.
> 모노크롬 + 화이트스페이스 + 단어 분할 헤딩 + 검정 원형 강조.

## 1. 컬러 토큰

| 토큰 | HEX | 용도 |
|------|-----|------|
| `--ink` | `#000000` | 메인 헤딩, 검정 원형, 클로징 배경 |
| `--ink-soft` | `#2D2D2D` | 다크 박스, 보조 헤딩 |
| `--paper` | `#FFFFFF` | 기본 배경 |
| `--cream` | `#F5F0E8` | 액센트 배경(섹션 구분, 카드) |
| `--rule` | `#E5E5E5` | 테두리, 점선, placeholder 박스 |
| `--muted` | `#8A8A8A` | 보조 텍스트, 페이지 번호 |
| `--body` | `#1A1A1A` | 본문 텍스트 |

**사용 비율(60/30/10)**: 화이트 60% · 블랙 30% · 크림/그레이 10%.

**브랜드 옵션 컬러**: 김진수 작가 브랜드 사용 시 액센트만 골드 `#C9A961`로 치환(검정 원형 유지).

## 2. 타이포그래피

| 요소 | 폰트 | 크기 | 두께 | 행간 |
|------|------|------|------|------|
| 단어 분할 헤딩 | Pretendard / Noto Sans KR | 80-110pt | 900 (Black) | 0.95 |
| 슬라이드 제목 | Pretendard | 36-44pt | 700 | 1.1 |
| 섹션 헤더 | Pretendard | 20-24pt | 600 | 1.2 |
| 본문 | Pretendard | 14-18pt | 400 | 1.5 |
| 페이지 번호 | Pretendard | 10-12pt | 400 | 1.0 |
| 캡션/출처 | Pretendard | 9-11pt | 300 | 1.3 |

**핵심 규칙**:
- 헤딩은 **2-3음절 단위로 줄바꿈**: "Over / view", "개 / 요", "전 / 략"
- 본문은 무조건 left-align, 헤딩만 left 또는 center
- italic 사용 금지(절제된 톤 유지)

## 3. 그리드 & 여백

표준 16:9 슬라이드(13.33" × 7.5") 기준:

| 영역 | 값 |
|------|----|
| 상단 여백 | 0.7" |
| 하단 여백 | 0.7" |
| 좌우 여백 | 0.8" |
| 헤딩-본문 간격 | 0.5" |
| 카드 간 간격 | 0.3" |

**12컬럼 그리드** 사용. 좌 6 / 우 6 또는 좌 4 / 우 8 분할 빈도 높음.

## 4. 시그니처 비주얼 모티프

### 4-1. 검정 원형(Black Circle)
- 지름 1.8-2.4"
- 단어 1개를 흰색 텍스트로 중앙 배치 ("Simple", "AI", "GEO")
- 슬라이드당 1개만, 우상단 또는 중앙 우측에 배치
- 강조 요소가 분산되지 않도록 슬라이드 1장당 1회만 사용

### 4-2. 단어 분할 헤딩(Split Word Heading)
- 영문: "Over / view", "Con / tents", "Sum / mary"
- 한글: "개 / 요", "내 / 용", "정 / 리", "전 / 략"
- 슬래시(/)는 시각적 분할 표시일 뿐 실제로는 줄바꿈으로 구현

### 4-3. 점선 Placeholder 박스
- 1px 점선(`#E5E5E5`)
- "Click to add text" 형태 또는 실제 내용 박스
- 카드 구분 시 채움색 없이 점선만으로 영역 표시

### 4-4. 우상단 페이지 번호
- 위치: 우상단 0.3" × 0.3" 안쪽
- 형식: "21" (앞 0 없음, 두 자리)
- 색상: `--muted`

## 5. 슬라이드 레이아웃 7종

### L1. 타이틀 슬라이드
```
┌─────────────────────────────────┐
│                                 │
│   [큰 헤딩 2단 분할]              │
│                                 │
│   강사명 · 날짜                   │
│                                 │
└─────────────────────────────────┘
```

### L2. Overview / 목차 (좌 헤딩 + 우 콘텐츠)
```
┌─────────────────────────────────┐
│ Over           ●Simple   [21]   │
│ view                            │
│                                 │
│ [point box]    AS-IS|TO-BE|GOAL │
└─────────────────────────────────┘
```
- 좌측: 분할 헤딩
- 중앙: 검정 원형(키워드 1개)
- 우측: 3단 비교 박스(AS-IS / TO-BE / GOAL)

### L3. 개념 설명 (단일 메시지)
```
┌─────────────────────────────────┐
│ 제목                       [21] │
│                                 │
│ 핵심 한 문장 결론형               │
│ 보조 설명 2-3줄                   │
│                                 │
└─────────────────────────────────┘
```
- 한 슬라이드 한 메시지 원칙(v2 계승)
- 제목은 결론 형태("AI는 도구가 아니다")

### L4. 3단 비교 (AS-IS / TO-BE / GOAL)
```
┌─────────────────────────────────┐
│ 제목                       [21] │
│                                 │
│ ┌───────┬───────┬───────┐       │
│ │ AS-IS │ TO-BE │ GOAL  │       │
│ │       │       │       │       │
│ └───────┴───────┴───────┘       │
└─────────────────────────────────┘
```
- 박스 3개 동일 너비
- 헤더 라벨 위에 위치, 본문은 점선 박스 안

### L5. 원형 강조 (Circle Emphasis)
```
┌─────────────────────────────────┐
│ 제목                       [21] │
│                                 │
│  본문 좌측        ●키워드         │
│                                 │
└─────────────────────────────────┘
```
- 좌측 본문 60%, 우측 검정 원형 40%
- 원형 안에 핵심 단어 1개

### L6. 데이터/스탯 (Stat Callout)
```
┌─────────────────────────────────┐
│ 제목                       [21] │
│                                 │
│   87%   라벨 설명                │
│   (90pt) (16pt)                  │
│                                 │
└─────────────────────────────────┘
```
- 큰 숫자 80-110pt, 라벨 14-16pt
- 출처는 슬라이드 하단 9pt

### L7. 클로징 / 브랜딩
```
┌─────────────────────────────────┐
│■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■│
│■                              ■│
│■        ●Your Brand           ■│
│■                              ■│
│■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■│
│ LOCATION    │   CONTACT         │
└─────────────────────────────────┘
```
- 상단 80% 풀블랙(`--ink`)
- 중앙에 검정 원형(`--ink-soft` 미세 대비) + 흰색 텍스트
- 하단 20% 화이트, LOCATION / CONTACT 2단

## 6. 절대 금지

- ❌ 컬러 3개 이상 동시 사용(블랙/화이트/크림 외)
- ❌ 그라데이션, 그림자, 글로우 효과
- ❌ 이모지, 데코 라인, 풀와이드 컬러바
- ❌ 센터 정렬 본문(센터는 헤딩만)
- ❌ italic, underline
- ❌ 한 슬라이드 다중 메시지(한 슬라이드 한 메시지 엄수)

## 7. 자가 검수 체크리스트

빌드 후 다음 7개 항목을 모두 통과해야 한다:

- [ ] 컬러는 블랙/화이트/크림 + 1액센트 이내인가
- [ ] 모든 본문이 left-align인가
- [ ] 페이지 번호가 모든 슬라이드 우상단에 있는가
- [ ] 시그니처 모티프(검정 원형 / 단어 분할 / 점선 박스) 중 1개 이상 사용했는가
- [ ] 결론형 제목인가(개념 나열형 아닌)
- [ ] 폰트는 1-2종 이내인가
- [ ] 슬라이드당 메시지 1개 원칙을 지켰는가

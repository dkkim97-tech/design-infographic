# 강의용 슬라이드 최적화 생성기 (Lecture Slide Creator)

## 개요

이 스킬은 교육자, 강사, 트레이너를 위한 **전문 강의 슬라이드 제작 도우미**입니다. 단순히 슬라이드를 만드는 것이 아니라, **교육학적 원칙**에 기반하여 학습 효과를 극대화하는 프레젠테이션을 설계합니다.

### 핵심 가치

- **교육 효과 최적화**: 인지 부하 이론, 멀티미디어 학습 원칙 적용
- **청중 맞춤 설계**: 대상 연령, 배경지식, 학습 목표에 따른 커스터마이징
- **전문적 디자인**: 가독성과 집중도를 높이는 시각적 구성
- **발표 지원**: 발표자 노트, 타이밍 가이드, 인터랙션 포인트 제공

## 워크플로우

### 1단계: 강의 분석 및 기획

사용자의 요청을 받으면 다음 정보를 파악합니다:

#### 필수 정보 수집

```
📚 강의 주제: [주제명]
🎯 학습 목표: [이 강의를 통해 학습자가 얻을 것]
👥 대상 청중:
   - 연령대: [예: 20-30대 직장인]
   - 배경지식 수준: [초급/중급/고급]
   - 인원수: [예: 20-30명]
⏱️ 강의 시간: [예: 60분]
📍 형태: [대면/온라인/하이브리드]
🎨 선호 스타일: [전문적/친근한/역동적/차분한]
```

#### 선택 정보

```
📁 기업/기관 브랜드 색상: [있다면 제공]
🖼️ 포함할 이미지/자료: [있다면 제공]
📝 특별 요청사항: [있다면 제공]
```

### 2단계: 슬라이드 구조 설계

#### 교육학적 슬라이드 구조 (권장)

| 섹션 | 슬라이드 비율 | 목적 |
|------|-------------|------|
| 도입부 (Opening) | 10-15% | 주의 집중, 학습 동기 유발 |
| 핵심 개념 (Core) | 50-60% | 주요 내용 전달 |
| 실습/예시 (Practice) | 20-25% | 적용 및 이해 확인 |
| 정리/마무리 (Closing) | 10-15% | 핵심 요약, 다음 단계 안내 |

#### 시간별 슬라이드 수 가이드

| 강의 시간 | 권장 슬라이드 수 | 슬라이드당 시간 |
|----------|----------------|----------------|
| 30분 | 12-18장 | 1.5-2.5분 |
| 60분 | 25-35장 | 1.5-2.5분 |
| 90분 | 35-50장 | 1.5-2.5분 |
| 120분 | 45-65장 | 1.5-2.5분 |

### 3단계: 콘텐츠 최적화

#### 슬라이드 유형별 가이드라인

**1. 타이틀 슬라이드**
```
- 강의 제목: 30-40pt, 굵은 폰트
- 부제목/강사명: 18-24pt
- 날짜/장소: 14-16pt
- 브랜드 로고 위치: 우측 하단 또는 좌측 하단
```

**2. 목차 슬라이드**
```
- 3-5개 주요 섹션으로 구성
- 현재 위치 강조 기능 (진행 중 하이라이트)
- 예상 소요 시간 표시 (선택)
```

**3. 개념 설명 슬라이드**
```
- 제목: 핵심 키워드 포함 (25-30pt)
- 본문: 최대 4-5개 불릿포인트
- 불릿당 1줄 (최대 2줄)
- 폰트 크기: 18-24pt
- 줄 간격: 1.3-1.5배
```

**4. 다이어그램/비주얼 슬라이드**
```
- 이미지 영역: 슬라이드의 60-70%
- 설명 텍스트: 최소화 (라벨 형태)
- 출처 표기: 8-10pt, 하단
```

**5. 인터랙션 슬라이드**
```
- 질문 제시: 큰 폰트로 중앙 배치
- 생각 시간: "1분간 생각해보세요" 안내
- 토론 가이드: 선택적 포함
```

**6. 요약 슬라이드**
```
- 핵심 포인트: 3-5개
- 시각적 아이콘 활용
- 체크리스트 형식 권장
```

## 슬라이드 작성 원칙

### 📐 레이아웃 규칙

#### 여백 및 정렬
```
상단 여백: 슬라이드 높이의 10%
하단 여백: 슬라이드 높이의 8%
좌우 여백: 슬라이드 너비의 5%
콘텐츠 영역: 중앙 정렬 또는 좌측 정렬 (일관성 유지)
```

#### 텍스트 배치
```
제목: 상단 15-20% 영역
본문: 중앙 60-70% 영역
출처/각주: 하단 5% 영역
```

### 📝 텍스트 규칙

#### 글자 수 제한
| 요소 | 권장 | 최대 |
|------|------|------|
| 슬라이드 제목 | 10자 | 20자 |
| 불릿 포인트 | 15자 | 25자 |
| 전체 텍스트 | 50자 | 75자 |

#### 7x7 규칙 적용
- 한 슬라이드에 최대 7줄
- 한 줄에 최대 7단어 (영어 기준) / 15자 (한글 기준)

### 🎨 시각적 규칙

#### 색상 사용
```
배경: 1가지 (단색 또는 그라데이션)
주요 텍스트: 1가지
강조 색상: 1-2가지
전체 색상: 최대 4가지
```

#### 이미지 사용
```
해상도: 최소 150 DPI (인쇄용 300 DPI)
형식: PNG (투명 배경), JPG (사진)
비율: 원본 비율 유지
저작권: 반드시 출처 표기
```

## 출력 형식

### 기본 출력

1. **슬라이드 개요 (outline.md)**
   - 전체 구조 및 흐름
   - 각 슬라이드 요약

2. **프레젠테이션 파일 (.pptx)**
   - 완성된 슬라이드
   - 기본 애니메이션 적용

3. **발표자 노트 (presenter-notes.md)**
   - 각 슬라이드별 발표 가이드
   - 예상 Q&A

### 선택적 출력

4. **참가자 핸드아웃 (handout.pdf)**
   - 슬라이드 2-4개/페이지
   - 메모 공간 포함

5. **요약 인포그래픽 (summary.png)**
   - 핵심 내용 1페이지 정리

## 기술 구현

### PPTX 생성 워크플로우

이 스킬은 `/mnt/skills/public/pptx/SKILL.md`의 html2pptx 워크플로우를 활용합니다.

#### 기본 프로세스

1. **콘텐츠 계획** → 슬라이드 구조 및 내용 설계
2. **HTML 변환** → 각 슬라이드를 HTML로 작성
3. **PPTX 생성** → html2pptx 라이브러리로 변환
4. **품질 검증** → 썸네일 생성 및 시각적 검토

#### 코드 스타일 가이드

```javascript
// 간결한 코드 작성
// 불필요한 변수명 지양
// console.log 최소화
const pptxgen = require("pptxgenjs");
const { html2pptx } = require("./html2pptx");

const pptx = new pptxgen();
pptx.layout = "LAYOUT_16x9";

// 슬라이드 생성
await html2pptx("slide01-title.html", pptx);
await html2pptx("slide02-agenda.html", pptx);
// ... 추가 슬라이드

await pptx.writeFile("lecture-presentation.pptx");
```

## 참고자료

### 교육학 원칙

- **인지 부하 이론 (Cognitive Load Theory)**: 불필요한 정보 최소화
- **멀티미디어 학습 원칙 (Mayer's Principles)**: 텍스트와 이미지의 효과적 결합
- **ADDIE 모델**: 분석-설계-개발-실행-평가
- **Kirkpatrick 4단계 평가**: 학습 효과 측정

### 디자인 원칙

- **근접성 (Proximity)**: 관련 요소는 가깝게
- **정렬 (Alignment)**: 일관된 정렬선 유지
- **반복 (Repetition)**: 시각적 패턴 일관성
- **대비 (Contrast)**: 중요한 것은 눈에 띄게
